Additionally it can be used for unescaped string literals; this is particularly relevant with path constants, e.g. instead of path = "c:\\\\temp\\\\somefile.txt" you can write path = @"c:\\temp\\somefile.txt". If x is a variable, @x is another name for the same variable.But the @ prefix does have a use, as you\'ve discovered - you can use it to clarify variables names that C# would otherwise reject as illegal.
