There is actually a fairly detailed entry in the Dev Guide, consider reading it for details. In short, fragments live inside activities, and each activity can host many fragments. Like activities, they have a specific lifecycle, unlike activities, they are not top-level application components. Advantages of fragments include code reuse and modularity (e.g., using the same list view in many activities), including the ability to build multi-pane interfaces (mostly useful on tablets). The main disadvantage is (some) added complexity. You can generally achieve the same thing with (custom) views in a non-standard and less robust way. A Fragment is a piece of an application\'s user interface or behavior that can be placed in an Activity which enable a more modular activity design.
