According to the MySQL documentation: You cannot reset the counter to a value less than or equal to any that\n have already been used. For MyISAM, if the value is less than or equal\n to the maximum value currently in the AUTO_INCREMENT column, the value\n is reset to the current maximum plus one. For InnoDB, you can use\n ALTER TABLE ... AUTO_INCREMENT = value as of MySQL 5.0.3, but if the\n value is less than the current maximum value in the column, no error\n occurs and the current sequence value is not changed.In essence, you can only alter AUTO_INCREMENT to increase the value of the autoincrement column, not reset it to 1, as the OP asks in the second part of the question.
