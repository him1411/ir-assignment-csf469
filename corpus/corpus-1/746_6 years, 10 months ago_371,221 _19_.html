I just want to say something like git diff can show you the difference between two commits:Or, equivalently:Using the latter syntax, if either side is HEAD it may be omitted (e.g. master.. compares master to HEAD).You may also be interested in mybranch...master (from git diff docs):This form is to view the changes on the branch containing and up to the second , starting at a common ancestor of both . git diff A...B is equivalent to git diff $(git-merge-base A B) B.In other words, this will give a diff of changes in master since it diverged from mybranch (but without new changes since then in mybranch).In all cases, the -- separator before the file name indicates the end of command line flags. This is optional unless Git will get confused if the argument refers to a commit or a file, but including it is not a bad habit to get into.
