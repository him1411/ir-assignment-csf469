I know this question is fully answered, but I\'d like to summarize the (few) safe steps to undo a git commit for those (like me) who were in panic after a bad commit:go into the source directory and make a backup of your code with command:tar czvf code.tgz *display the git commit list with the command:git logget the "sha1-commit-id" of the commit under your unwanted commit, this is the id of the commit before yours (the git log lists the more recent commit first)revert back the unwanted commit with the commandgit reset --hard "sha1-commit-id"*you can check you did it correctly by issuing anoter git log command*get back your uncommited code with the command:tar xzvf code.tgzThe tar commands are used to get sure your hard work does not get wiped by mistake, as the git reset command will delete your working directory changes, so step 1 and 5 are used to stash the local changes.
