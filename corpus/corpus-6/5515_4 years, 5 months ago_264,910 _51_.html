In slow Internet connections you\'ll have to wait a lot until the request is sent and also wait for the file to download. It isn\'t like an instant "click" => "popup" => "download start". It\'s more like "click" => "wait until data is sent" => "wait for response" => "download start" which makes it appear the file double its size because you\'ll have to wait for the request to be sent through AJAX and get it back as a downloadable file.If you\'re working with small file sizes <1MB you won\'t notice this. But as I discovered in my own app, for bigger file sizes it is almost unbearable.My app allow users to export images dynamically generated, these images are sent through POST requests in base64 format to the server (it is the only possible way), then processed and sent back to users in form of .png, .jpg files, base64 strings for images +1MB are huge, this force users to wait more than necessary for the file to start downloading. In slow Internet connections it can be really annoying.My solution for this was to temporary write the file to the server, once it is ready, dynamically generate a link to the file in form of a button which changes between "Please wait..." and "Download" states and at the same time, print the base64 image in a preview popup window so users can "right-click" and save it. This makes all the waiting time more bearable for users, and also speed things up.Update Sep 30, 2014:Months have passed since I posted this, finally I\'ve found a better approach to speed things up when working with big base64 strings.
