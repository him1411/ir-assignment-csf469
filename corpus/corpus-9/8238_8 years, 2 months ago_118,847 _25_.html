Maven absolutely was designed for this type of dependency.mvn package won\'t install anything in your local repository it just packages the project and leaves it in the target folder.Do mvn install in parent project (A), with this all the sub-modules will be installed in your computer\'s Maven repository, if there are no changes you just need to compile/package the sub-module (B) and Maven will take the already packaged and installed dependencies just right.You just need to a mvn install in the parent project if you updated some portion of the code.If you have previously run mvn install on project B it will have been installed to your local repository, so when you build package A Maven can resolve the dependency.
