What\'s the difference between event.stopPropagation() and event.stopImmediatePropagation()?stopPropagation will prevent any parent handlers from being executed stopImmediatePropagation will prevent any parent handlers and also any other handlers from executing Quick example from the jquery documentation: From the jQuery API:In addition to keeping any additional\n handlers on an element from being\n executed, this method also stops the\n bubbling by implicitly calling\n event.stopPropagation(). To simply\n prevent the event from bubbling to\n ancestor elements but allow other\n event handlers to execute on the same\n element, we can use\n event.stopPropagation() instead.Use\n event.isImmediatePropagationStopped()\n to know whether this method was ever\n called (on that event object).In short: event.stopPropagation() allows other handlers on the same element to be executed, while event.stopImmediatePropagation() prevents every event from running.Added a small example on jsfiddle to demonstrate how both these propagation stoppages work.
