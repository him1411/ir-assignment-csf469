I don\'t think it tells you anything different from what others have said. But it does provide the official reference.keystore/truststoreA keystore is a database of key material. Key material is used for a variety of purposes, including authentication and data integrity.\n Various types of keystores are available, including PKCS12 and\n Oracle\'s JKS.Generally speaking, keystore information can be grouped into two categories: key entries and trusted certificate entries. A key entry\n consists of an entity\'s identity and its private key, and can be used\n for a variety of cryptographic purposes. In contrast, a trusted\n certificate entry contains only a public key in addition to the\n entity\'s identity. Thus, a trusted certificate entry cannot be used\n where a private key is required, such as in a\n javax.net.ssl.KeyManager. In the JDK implementation of JKS, a keystore\n may contain both key entries and trusted certificate entries.A truststore is a keystore that is used when making decisions about what to trust. If you receive data from an entity that you\n already trust, and if you can verify that the entity is the one that\n it claims to be, then you can assume that the data really came from\n that entity.An entry should only be added to a truststore if the user trusts that entity. By either generating a key pair or by importing a\n certificate, the user gives trust to that entry. Any entry in the\n truststore is considered a trusted entry.It may be useful to have two different keystore files: one containing just your key entries, and the other containing your\n trusted certificate entries, including CA certificates. The former\n contains private information, whereas the latter does not. Using two\n files instead of a single keystore file provides a cleaner separation\n of the logical distinction between your own certificates (and\n corresponding private keys) and others\' certificates. To provide more\n protection for your private keys, store them in a keystore with\n restricted access, and provide the trusted certificates in a more\n publicly accessible keystore if needed.First and major difference between trustStore and keyStore is that trustStore is used by TrustManager to determine whether remote connection should be trusted, keyStore is used from KeyManager deciding which authentication credentials should be sent to the remote host for authentication during SSL handshake.Another difference is that keyStore theoretically contains private keys required only if you are running a Server in SSL connection or you have enabled client authentication on server side and on the other hand trustStore stores public key or certificates from CA (Certificate Authorities) which are used to trust remote party or SSL connection.
