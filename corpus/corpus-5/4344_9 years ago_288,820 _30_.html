However, its pure-java version,\n which totally translates c/c++ codes of SQLite into Java, is\n significantly slower compared to its native version, which uses SQLite\n binaries compiled for each OS (win, mac, linux).To use the native version of sqlite-jdbc, user had to set a path to\n the native codes (dll, jnilib, so files, which are JNDI C programs) by\n using command-line arguments, e.g., -Djava.library.path=(path to the\n dll, jnilib, etc.), or -Dorg.sqlite.lib.path, etc. This process was\n error-prone and bothersome to tell every user to set these variables.\n Our SQLiteJDBC library completely does away these inconveniences.Another difference is that we are keeping this SQLiteJDBC libray\n up-to-date to the newest version of SQLite engine, because we are one\n of the hottest users of this library.
